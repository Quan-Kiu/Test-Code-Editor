# Decision Records

This folder stores why important decisions were made.

Create a decision record when choosing or changing:

- architecture
- database schema
- auth/security
- payment provider
- major dependency
- API contract
- validation strategy
- deployment strategy
- production operations strategy

Decision records should be small and specific.

## Product-quality decisions

Also create or update a decision record when the choice changes target quality level, role applicability, plan strategy, UX/composition direction, performance budget, security/privacy posture, rollback strategy, or release readiness. Link the relevant role or gate from `docs/role-gate-quality-map.md` and the plan-quality reasoning from `docs/plan-quality.md`.
