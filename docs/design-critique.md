# Design Critique Checklist

Use this after AI creates UI.

## 1. First impression

- [ ] Does the screen have a clear focal point?
- [ ] Does it feel like a real product, not a generic template?
- [ ] Does the mood match `DESIGN.md`?
- [ ] Is there too much decoration?
- [ ] Is there too little personality?

## 2. Hierarchy

- [ ] Is the primary action obvious?
- [ ] Are headings, body, captions visually distinct?
- [ ] Can the user scan the page in 5 seconds?

## 3. Components

- [ ] Buttons have clear states.
- [ ] Inputs have labels.
- [ ] Error states are visible.
- [ ] Empty states guide action.
- [ ] Loading states do not feel broken.

## 4. Accessibility

- [ ] Text contrast is readable.
- [ ] Focus states are visible.
- [ ] Keyboard navigation is possible.
- [ ] Touch targets are comfortable.


## 5. Human-level composition

- [ ] Component rhythm feels consistent across repeated cards, controls, panels, and text groups.
- [ ] Spatial balance feels intentional across left/right, top/bottom, foreground/background, and negative space.
- [ ] Center-content preservation keeps the main subject/action inside a safe focal zone on mobile, desktop, scroll, overlays, motion, and camera movement where applicable.
- [ ] Visual dominance belongs to the intended subject, scene, CTA, or player goal instead of debug UI, HUD, labels, panels, or decoration.
- [ ] The eye path in the first three seconds moves from subject to meaning to next action.
- [ ] Visual noise does not compete with the main task or scene.
- [ ] A user/player can understand the goal and next step without developer notes.
